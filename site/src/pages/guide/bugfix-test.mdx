# Bug Fix Test Page [Visual test for shell copy, code group height, and shell annotations]

This page reproduces three bugs to verify fixes.

## Bug 1: Multiline bash copy

Clicking the copy button on a multiline `\`-continuation command should copy **all** continuation lines, not just the first.

```bash
$ tempo node \
  --follow \
  --http --http.port 8545 \
  --http.api eth,net,web3,txpool,trace
```

```bash
$ sudo tee /etc/systemd/system/tempo.service > /dev/null <<EOF \
  --datadir /data \
  --follow \
  --http \
  --http.addr 0.0.0.0
```

Single-line command (should still work normally):

```bash
$ echo hello world
```

    primitives::{address, Address, U256},

            .abi_encode()

## Bug 3: `[!code hl]` with bash backslash continuations

The `# [!code hl]` annotation should be **stripped** from the output and the line should be highlighted. Previously the annotation was left visible because `\` before `#` prevented Shiki from recognizing it as a comment.

```bash
tempo node \
  --follow \ # [!code hl]
  --http --http.port 8545 \
  --http.api eth,net,web3,txpool,trace
```

Mixed annotations — highlight and focus:

```bash
docker run \
  --name tempo \ # [!code hl]
  -p 8545:8545 \ # [!code hl]
  -v /data:/data \ # [!code focus]
  tempo:latest
```

Normal `# [!code hl]` (non-continuation) should still work:

```bash
echo "this line is highlighted" # [!code hl]
echo "this line is not"
```
