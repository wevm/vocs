# Project Structure

Vocs uses a file-based routing system. Here's the typical structure of a Vocs project.

## Directory Layout

:::file-tree
- my-docs/
  - src/
    - pages/
      - index.mdx [Home page (/)]
      - guide/
        - intro.mdx [/guide/intro]
        - setup.mdx [/guide/setup]
      - api/
        - config.mdx [/api/config]
  - public/ [Static assets]
  - vocs.config.ts [Vocs configuration]
  - package.json
:::

## Pages Directory

The `src/pages/` directory contains all your documentation pages. Each `.mdx` or `.md` file becomes a route.

## Configuration

The `vocs.config.ts` file is where you configure your site's title, description, sidebar, and more.

## Static Assets

Place static files like images in the `public/` directory. They will be served at the root path.
